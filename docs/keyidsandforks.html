The basic idea behind a keyid, is its a alphanumeric id, that either exists in the keylist or doesn't.  This creates a form of Boolean logic.

forks belong in the "fork" section of pages.

there is a special id that you should never try to use, as its used as a "null/fallback" keyid, and will be ignored.
"0"

the idea behind "keyid forks" or "forks" for short, is you are "forking" the keyids in different ways, and are causing various events to occur.

Desutezeoid's test/example program includes many examples of these forks in action, also, for using core objects with keyids, see the core objects documentation.

List of Keyid forks


XML code of fork
 Description
<timeout keyid="prototime" seconds="3.5" post="soundfx1"/>
 The timeout fork removes keyid specified with "keyid attribute after "seconds" has passed. if "post" is specified, that keyid will be added after the first "times out"
<batchset keyid="switchinvert" set="2">
      <k keyid="switch1"/>
      <k keyid="switch2"/>
</batchset>
 batchset removes the keyid specified in "keyid" when triggered, and will either add (set="1"), remove (set="0"), or invert (set="2") the keyids specified in each "k" child element's "keyid" attribute.
<sound keyid="soundfx1" sound="mip.ogg"/>
 sound removes the keyid specified in the "keyid" attribute, and plays the sound file specified by "sound"
<triggerlock keyid="alpha3" lock="soundfx1lock1" trigger="soundfx1"/>
 triggerlock doesn't remove the "keyid" attribute keyid, but instead adds "lock" attribute keyid thus preventing it from triggering "trigger" more than once unless "lock" attribute keyid is removed
<music keyid="mustrkload" track="BGM1.ogg" />
<music keyid="mustrkstop" stop="1"/>
<music keyid="mustrkplay" play="1"/>
 music is similar to the sound fork, but controls the Background music. setting stop="1" will stop the music, setting play="1" will play the most recently played track, and setting "track" to a sound file will play that sound file on loop as the background music, as if it was triggered by a page's music settings.
<pagejump keyid="pagejumpmenu1" page="about.xml"/>
 pagejump removes its "keyid" attribute keyid and switches the Page XML to that specified in the "page" attribute. its basically a fork equivalent to the "iref" act type.
<batchtrig keyid="alpha">
      <k if="1" keyid="switch1"/>
      <k if="0" keyid="switch2"/>
</batchtrig>
 batchtrig is the opposite to batchset, in that it adds "keyid" attribute keyid if ALL the conditions laid out in the "k" child tags is met. if="1" means the k tag's keyid must be present, if="0" means it must not be present.
<ortrig keyid="alpha4">
       <k if="1" keyid="switch1"/>
       <k if="1" keyid="switch2"/>
</ortrig>
 ortrig is exactly the same as batchtrig except 1 thing: only one of the k tags needs its condition met, hence the tag name: ortrig
<uipop keyid="uidialog3" msg="This Dialog is red." BGCOLOR="#AA0000" FGCOLOR="#FFFFFF" />

<uipop keyid="uidialog4" msg="This Dialog is BIG." BGCOLOR="#00AA00" FGCOLOR="#FFFFFF" textsize="70" />

<uipop keyid="uidialog2" msg="Hello, this is a yes or no dialog." ynflag="1" nokey="nokeyyndiag" yeskey="yeskeyyndiag" />

<uipop keyid="uidialog5" msg="This Dialog has an image." BGCOLOR="#0000AA" FGCOLOR="#FFFFFF" textsize="25" img="testimage300.png" />

 uipop is a useful fork to present the user with a popup dialog.
you can overwrite the ENGSYSTEM.XML settings for colors and text size using "FGCOLOR", "BGCOLOR", and "textsize"
you can specify an image using the "img" attribute, if you specify ynflag="1", specify nokey and yeskey to prompt the user with a yes/no question.

"msg" specifies the message, and like many other forks its triggered by the keyid specified in "keyid", and removes that keyid when triggered.
<uimenu keyid="uimenu1" BGCOLOR="#0000AA" FGCOLOR="#FFFFFF" textsize="25">
       <item con="This is a menu." noact="1"/>
       <item con="-------------------" noact="1"/>
           
       <item con="play sound" keyid="soundfx1" stay="1"/>
       <item con="run test 1" keyid="pagejumpmenu1" stay="1"/>
       <item con="test dialog with image" keyid="uidialog5" stay="1"/>
       <item con="play music" keyid="mustrkplay" stay="1"/>
       <item con="stop music" keyid="mustrkstop" stay="1"/>
       <item con="close menu"/>
</uimenu>
 uimenu is another popup fork, but presents the user with a menu. each menu item is a child element called "item",  the "con" attribute defines the item's text, specity "keyid" to add a keyid on-click, and use stay="1" to keep the menu open on click.

also, for unclickable items, use noact="1"
