core objects are the main way of showing something on the screen.

naturally they belong in the "core" section of pages.

they come in two classes:

clickable core objects like "img", "label", and "box"
UNclickable core objects like "text"

clickable core objects have a special child element called "act" the act tag has 2 attributes: "type" and "ref"
"type" referrs to what Desutzeoid should do when the item is clicked:
type="none" means do absolutely nothing. no sounds, no onclick keyids.
type="key" means do everything but parse page switching.
type="iref" does everything, including use the value of "ref" as the page XML filename to switch to.
type="quit" presents the quit dialog.


clickable core objects also have some common attributes:
("0" is used a as a placeholder. like it states in keyids and forking, "0" is not a real keyid and is ignored by Desutezeoid.)
keyid="0" add a keyid on click
takekey="0" take a keyid on click
onkey="0" only activate this core object if a keyid is present ("text" supports this)
offkey="0" only activate this core object if a keyid NOT present ("text" supports this)
hoverkey="onhov1"  This nifty keyid attribute causes the specified keyid to be actively removed if the mouse is not over the object, and vice versa if it is.
sfxclick="1" set this to "1" if you want to use an on-click sound.
sound="mip.ogg" this is used to specify the on-click sound when its enabled with the sfxclick attribute.

 Labels use pygame's builtin text rendering capacity to, well, generate labels. Use BGCOLOR and FGCOLOR to specify the colors, also set transp="1" to have text with a transparent background. also use "size" to specify text size

the con child element is used to define the text to be used as the label.
<text x="10" y="30" size="20" transp="1" FGCOLOR="#000000">This is an
example of the text core object.
interesting eh?</text>
 this nifty text tag supports the color, transparency, and text size of the label tag, but is used to show paragraphs of text. though, it does not support click events.
<img x="300" y="100" onkey="timeloop1">
       <con>no-tr-animtest1a.png</con>
       <act type="none"/>
</img>
 ahh the img tag. its by far the most useful of the bunch, as any raster image on screen is shown using it.

the con attribute specifies the filename.

speaking of filenames, if the filename is prefixed with "no-tr", or the filettype is a jpeg image, Desutezeoid will optimize the image with alpha disabled, else it will optimize it with alpha enabled.

naturally there are some extra attributes:

vscroll="1" scrolls the image vertically by the specified number of pixels each frame. use negative values for the opposing direction
hscroll="1" scrolls the image horizontally by the specified number of pixels each frame. use negative values for the opposing direction.

mouseh="2" and mousev="2":
that nifty pair of horizontal and vertical attributes allows an img tag to move based on mouse coordinates.
1= follow mouse, aligned to top left corner of image
2= follow mouse, aligned to center of image,
3= follow a "screen inversion" of the mouse position, aligned to the center of the image. useful for wide panoramic views (the test program has some examples).

hovpic="1"
this nifty attribute enables a copy of the img con child tag called altcon, and uses it when the mouse is hovering over the img object.
<box x="300" y="200" sizex="40" sizey="60" COLOR="#FF0000" alpha="60" hovalpha="150" onhov="1" HOVCOLOR="#FF0000">
       <act type="none"/>
</box>
<box x="295" y="44" sizex="175" sizey="123" alpha="0" hoverkey="sometip" keyid="someclick">
       <act type="key"/>
</box>
 ahh the ever useful box. the primary use of this is creating "invisible" clickable core objects, but also has COLOR, alpha, hovalpha, and HOVCOLOR, to have some basic highlighting action.
set onhov="1" to enable the separate HOVCOLOR and hovalpha attributes.

setting alpha="0" disables COLOR altogether.

use sizex and sizey, to specify size.

tip: the image editor, gimp shows the position and size of rectangle selections in the tool settings tab. using this trick on your backdrop (if its at 0,0) can help in adding box objects.
